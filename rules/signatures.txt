# Custom content signatures (Snort/Suricata-style, simplified).
# Format:  name | threat_type | substring
#   name        = human-readable rule name
#   threat_type = one of the canonical threat types (drives CVSS scoring)
#   substring   = case-insensitive substring to match in the canonicalized payload
# Matched against the normalized payload (URL/unicode-decoded, lowercased).
#
# PRECISION POLICY: in Live (NIDS) mode a signature match is a DETERMINISTIC
# verdict that SKIPS the LLM, so only high-precision, essentially-never-benign
# patterns belong here. Broad/ambiguous patterns (e.g. "union select",
# "<script>", "/etc/passwd") are intentionally NOT signatures — they are left
# for the LLM to arbitrate so Live mode does not false-positive on legitimate
# web traffic. See the Threat Tagging & False-Positive Tuning note.
